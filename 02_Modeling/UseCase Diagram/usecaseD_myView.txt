1-3. 기본흐름
1) 사용자가 업로드한 포스트에 대한 목록이 사용자에게 보여진다.
2) 사용자가 자신의 포스트를 수정하는 경우,
  + 기존의 업로드된 포스트 목록 중 수정하고자 하는 포스트를 선택한다.
  + 선택된 포스트의 상단부의 수정/삭제 버튼 중, 수정 버튼을 선택한다.
  + 포스트에 대한 수정을 완료하였다면 하단부에 위치한 저장/취소 버튼 중 저장버튼을 선택한다.
3) 사용자가 자신의 포스트를 삭제하는 경우,
  + 기존의 업로드된 포스트 목록 중 삭제하고자 하는 포스트를 선택한다.
  + 선택된 포스트의 상단부의 수정/삭제 버튼 중, 삭제 버튼을 선택한다.
  + "삭제 하시겠습니까"라는 알림창에 대해 확인/취소 버튼중 확인을 선택한다.
4) 사용자가 새 포스트를 작성하는 경우,
  + 화면의 상단부에 위치한 새 포스트 작성 버튼을 선택한다.
  + 포스트 작성 페이지에서 포스트를 작성한 후, 하단부에 위치한 저장/취소 버튼 중 저장을 선택합니다.
1-4. 대안흐름
1) 사용자가 포스트 수정을 취소하고자 하는 경우,
  + 포스트 수정 페이지의 하단부에 위치한 저장/취소 버튼 중 취소 버튼을 클릭하여 포스트 목록 페이지로 돌아간다.
2) 사용자가 포스트 삭제를 취소하고자 하는 경우,
  + 삭제하고자 하는 포스트에서 삭제 버튼을 눌러, "삭제 하시겠습니까" 알림창이 떴을때 확인/취소 버튼 중 취소를 선택하여 포스트 확인 페이지로 돌아간다.
1-5. 예외흐름
1) 사용자가 포스트를 작성할때 제목을 입력하지 않은 경우,
  + "제목을 입력하세요"라는 알림창이 팝업으로 사용자에게 보여지며, 확인 버튼을 누르게 되면 포스트 작성 페이지로 돌아간다.
